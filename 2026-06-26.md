## WS-1008 board status 漂移复核 (CTO Codex, 09:45 CST)

**任务**: 复核 WS-966 / WS-963 出现 "agent run 自报已切状态, board 仍 in_progress" 的 P1 漂移。

**结论**: 这两条不是 "CLI 没有真调用" 或 "CLI 成功但服务端 silent swallow"。两条被点名的 completed run 都真实调用了 status CLI, 且工具结果返回成功。当前 board 状态来自后续状态变更 / 后续业务 run, 不是原 status 命令失败。

**证据**:

- WS-966 run `3407b169-fadd-4c5e-8cec-cfeac2ede0b3`: run message seq 76 调用 `multica issue status ed435a03-3dc0-45a4-8014-fc36bdf058f3 blocked`; seq 79 返回 `Issue WS-966 status changed to blocked.`。同 run seq 78 评论也已落盘。
- WS-966 后续 run `f919d3aa-a715-4c70-aef8-1be884bf797f` 在 2026-06-25 14:47 上海读取 board 时看到 status=`in_review`, updated_at=`2026-06-25T06:46:59Z`; 说明 WS-966 在初次 blocked 后还被新的业务进展推进过。当前 2026-06-26 09:45 复核时 board 又是 `in_progress`, updated_at=`2026-06-25T09:06:04Z`; CLI 暴露的 run/comment 历史没有显示这次 17:06 上海变更的发起者。
- WS-963 run `7414341a-ef26-4616-ad5b-b2a02fe48b91`: run message seq 56 调用 `multica issue status 0f9dddc3-5b16-4772-b3a2-7039115fa3f8 in_review`; seq 57 返回 `Issue WS-963 status changed to in_review.`。
- WS-963 后续 autopilot run `991c1d3f-8e4e-4faf-adaf-9934474bc934` 在 2026-06-25 18:05 上海审计失败后明确写入: `WS-963 due_date -> 2026-06-26 (status 保持 in_progress)`。所以 WS-963 当前 `in_progress` 是后续 audit 业务决策, 不是 12:05 上海那次 `in_review` 没生效。

**根因判断**:

1. 初始被点名的两个 run 没有 CLI 失败; 不能按 "agent 误以为说了一句话就生效" 归因。
2. 真问题是 lifecycle 口径混在一起: run final output 是单次执行结束时的 self-report; board 是全局 issue 当前状态, 会被后续 comment-trigger / autopilot / 人工变更覆盖。
3. 仍有一个平台观测缺口: CLI 暂无 issue status audit log, 因此 WS-966 2026-06-25 17:06 上海从 `in_review` / `blocked` 系列状态回到 `in_progress` 的具体 actor 无法用现有 CLI 定位。

**修复建议**:

- Runtime postflight 应只校验 "本 run 完成后的即时状态": 如果 final output 声称已把 assigned issue 切到 terminal/review 状态, runtime 立即 `issue get` 复核 board; 不一致时再 reconcile 或至少写 warning。这个机制能抓住同 run 的虚报, 但不能阻止数小时后的合法后续状态变更。
- Control-plane 侧应补 issue status audit log CLI, 至少返回 actor_type/actor_id/source_task_id/from_status/to_status/created_at, 否则无法追到 WS-966 17:06 上海这类无评论状态变更。
- 对 run_only autopilot 的 final output 要避免写成永久状态事实; 用 "本 run 已执行 X, 当前 board 为 Y" 替代 "已切 Y"。

**本轮限制**:

- 当前 WS-1008 工作区无 repo, `multica workspace get` 返回 `repos: []`; 本机可见的 `hermes-agent-webapi` / `hermes-workspace` 不是 Multica issue/runtime control-plane 源码。没有可安全 patch 的 runtime repo, 所以本轮不能落 runtime postflight 代码。
