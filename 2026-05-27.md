## 2. 5/27 sharp edge #29: macOS Container EPERM Symlink

**位置**: `~/.claude/projects/-Users-tangyuanjc/memory/feedback_macos_container_symlink_eperm_sharp_edge_0527.md` (8.3KB, 16:05 写)

**现象**: Codex Desktop / Hermes container 模式下, 跨容器边界做 symlink (软链接) 触发 EPERM (Operation not permitted), 没有 fallback 路径就硬挂.

**Root cause**: macOS 14+ 强化 sandbox, container fs (容器文件系统) 跟 host fs 之间 symlink 默认 deny, 即使 entitlement (权利标识) 配置过. 跟 sharp edge #26 (EINTR signal) #27 (self-modify) #28 (asar 自改) 形成 "macOS 容器化 4 条 sharp edge" 同主题集合.

**Fix recipe**:
1. 检测 EPERM: 如 fs.symlink 抛 EPERM → fallback 走 copy + manifest pointer
2. Container 边界配置: 同 container 内 symlink 可行, 跨 container 改 hardlink (硬链接) 或 file copy
3. 部署前 smoke test: 在目标机跑 `ln -s /tmp/a /tmp/b` 测一下, 失败就 alert (跟 sharp edge #27 同模式)

**跨 agent 适用**: 任何 agent 写涉及 fs.symlink 的代码必加 EPERM catch (即使开发机本地能跑通)
