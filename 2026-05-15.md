### 起源 (5/15 凌晨上海时间)

- JC 5/15 03:10 反馈: 网站打不开 (Cloudflare Error 1033) + 担心 Mac mini 崩溃

### 战略决策 (CSO)

**为什么是 VPS 不是 MacBook**: MacBook 移动设备 + 网络变 + 电池损耗, 不适合 always-on. VPS 99.9% uptime + 跟 Mac mini 解耦 + 公网延迟低 + $6/月成本可控.

**为什么 Vultr Tokyo $6/月 (vhp-1c-1gb)**: 不需 ICP 备案 (海外服务商) + 离中国近 (~50ms) + 域名已挂 Cloudflare 自带 CDN + JC 完全没概念用最简单的方案.

**为什么混合架构 (不是纯 VPS)**: 数据敏感 (内部 AI 资讯 + 内部投稿 + 8 人密码) 仍 Mac mini 写入, VPS 只做 read-only web frontend, 通过 rsync 同步 sqlite. 平衡 always-on + 数据敏感.

**为什么仍跑 dev mode (npm run dev)**: 看 Mac mini launchd plist 发现 ai-hotboard 一直跑 vite dev mode 不是 production build (设计债, vite SSR 没自定义 production server). VPS 跟 Mac mini 同样跑 npm run dev, 1GB RAM + 2.3GB swap 够用. 改 production server 留 W2 sprint.
