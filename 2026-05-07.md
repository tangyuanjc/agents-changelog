### 关键内容

**核心规则**: L4/L3 BLOCK 后 CSO 必须复审是不是 cross-context (跨场景) 误用铁律, **不机械执行**。

**为什么必须升 SOP**: L4 paranoid (偏执) 视角找盲点真有用, 但**也会跨场景误用** — 把 A 场景立的铁律套到 B 场景上 BLOCK (关键词匹配优先于场景判断). 没复审就执行 = 假阳性流到 JC = JC 多一轮推翻 = CSO 失职 (违反 5/6 立的 JC × CSO 分工铁律).

**CSO 复审 4 步流程** (L4/L3 BLOCK 必走):
1. **三原则过审** (透明 / 共生 / 不以人类意志为转移) — 不冲突大概率是误用
2. **找该铁律原始 memory** — 看原始场景 / 守护对象 / 解决问题
3. **判断同场景 (真盲点处理) or 跨场景 (撤回 + 公开 ack)** — 跨场景不端给 JC 浪费 cycle
4. **跨场景误用归档 ledger 案例** — 跟 paranoia 阶段错配并列校准维度, 升级 prompt 模板

**升级触发实测** (5/6 multica 议题 L4 BLOCK 实测):

  - P0-2 3 员工同时接入违反 single-variable — single-variable 是数据归因实验铁律不是工具采用场景
- JC 撤 P0-1 + P0-2, CSO 公开 ack 失误。
